Dashbird — California-focused weather icons (minimal set)
===========================================================

Goal: one raster (or SVG) file per *bucket* the app uses for the hero,
nothing else. Pick each asset from your Flaticon filtered searches
(style_id=8, author_id=1, word=weather); do not bulk-download pages.

Data source: Open-Meteo `weather_code` (WMO Weather interpretation codes WW).

-------------------------------------------------------------------
A) CONDITION ICONS — exactly 7 files (required before wiring the UI)
-------------------------------------------------------------------

Filename        WMO codes (Open-Meteo)     Typical / extreme CA context
--------------- ------------------------ --------------------------------
clear.png       0, 1                     Clear / mainly clear; heat-wave skies
partly.png      2                        Partly cloudy (marine layer mix, etc.)
cloudy.png      3                        Overcast / “June Gloom”–type deck
fog.png         45, 48                   Coastal & valley fog, rime fog
rain.png        51–67, 80–82             Drizzle → atmospheric river heavy rain;
                                       includes freezing drizzle/rain (56–57, 66–67)
snow.png        71–77, 85, 86            Sierra / mountain snow & snow showers
storm.png       95, 96, 99               Thunderstorms; hail codes 96/99 (hail
                                       forecast nuance is model-dependent)

The app maps every API code to one of these seven buckets (see
public/js/panels/weather-polygon.js `classify()`). No separate icon is
used for “wind only” — Open-Meteo does not emit a dedicated wind WW code
for current conditions in this table.

Wildfire smoke: the WW table above has no distinct “smoke” code. Smoke
episodes may sometimes resemble fog in modeled `weather_code`, or may not
map 1:1. If you later add AQI or another field, add a new bucket + file
then — not part of this minimal set.

-------------------------------------------------------------------
Download rule
-------------------------------------------------------------------

1. Identify the seven rows in section A (plus B if needed).
2. From Flaticon, choose *one* icon per filename; download that asset only.
3. Save under public/icons/weather/ using the exact filenames above.
4. Keep Flaticon attribution per their license (footer or README).

No other weather icon files are required for California coverage tied to
this API for the last few years: floods/AR → rain; mountain snow → snow;
tstorms/hail → storm; freezing precipitation in the model → same as rain
bucket for icon purposes.
