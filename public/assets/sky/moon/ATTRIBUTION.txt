moonnow.png (hero “moon now” glyph)
===================================

Watercolor-style **circular lunar rainbow / moonbow + waterfall** art (Gemini‑generated raster,
supplied by the dashboard owner).

`public/js/lib/moon-phase.js` **`moonPhaseIconUrl()`** always loads this asset for the hero moon
glyph. SunCalc‑derived phase name, illumination % tooltips, and `dataset.moonOctant` / `dataset.moonPngId`
still reflect the real lunation—they only swap the raster.

To restore octant‑accurate Wikimedia‑style discs, point **`moonPhaseIconUrl`** back at
`/assets/sky/moon/phase-{n}.png` (see octant table below).


Moon phase icons (phase-0.png … phase-7.png) — optional Wikimedia convention
==============================================================================

Legacy path when not using **`moonnow.png`**:

  • `getMoonPhaseIndex8(date)` → lunation octant 0…7 (SunCalc illumination).
  • `LUNATION_OCTANT_TO_MOON_PNG_ID[octant]` → which `phase-{id}.png` to load
    (default: identity so octant `k` uses `phase-k.png`).

Wikimedia Commons convention (Daniel Kmiec / User:Dannix, CC BY 3.0 —
https://creativecommons.org/licenses/by/3.0/deed.en): raster
`phase-N.png` should match **Moon_phase_N.svg** meaning:

  N 0 — New moon
  N 1 — Waxing crescent
  N 2 — First quarter
  N 3 — Waxing gibbous
  N 4 — Full moon
  N 5 — Waning gibbous
  N 6 — Last quarter
  N 7 — Waning crescent

If your PNGs are derived from that SVG set, retain reasonable credit. The
SVG set often had a blue instructional background removed before
rasterizing. If you replaced the art entirely, update this file and, if
file names no longer match octants 0…7, adjust `LUNATION_OCTANT_TO_MOON_PNG_ID`
only (do not reorder phase labels without updating that map).
